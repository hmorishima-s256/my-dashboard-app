# 04. 画面仕様

## 4.1 全体レイアウト

- 3段構成（ヘッダー / コンテンツ / フッター）
- ダークテーマ + 緑アクセント
- コンテンツ領域のみスクロール

### なぜこの構成か

- ヘッダー/フッターを固定的に認知させ、主要操作（同期・日付・設定）を常に触れる状態にするため

## 4.2 ヘッダー左（日付）

- 表示: `yyyy/mm/dd(曜)`
- 日付ボタンで入力パネルを開閉
- 右側に日付シェブロン
  - 左 `◀`: 前日
  - 右 `▶`: 翌日

### 日付入力パネル

- 入力欄: 年 / 月 / 日
- ボタン: `キャンセル` / `今日` / `OK`
- `dd` 入力欄の Enter は `OK` と同等
- バリデーションエラーは赤枠 + メッセージ

### なぜこのUIか

- 前後日の巡回が多いため、左右シェブロンを近接配置して操作量を減らすため

## 4.3 ヘッダー中央（タブ）

- `予定表` / `タスク`
- アクティブタブは緑グラデーション

### なぜこのUIか

- 現在位置（どちらの文脈で作業中か）を一目で分かるようにするため

## 4.4 ヘッダー右（同期 + 認証）

### 未ログイン

- `同期` ボタンは無効
- `Googleログイン` ボタン表示

### ログイン済み

- `同期` ボタン有効
- 固定人型アイコン表示
- アイコンクリックでメニュー表示（名前/メール/ログアウト）

### なぜこのUIか

- アイコン有無に依存すると見た目が揺れるため、固定アイコンでUI安定性を優先

## 4.5 予定表タブ

- 列: `日時` / `件名` / `カレンダー名`
- 日時セル:
  - 終日: `1day`
  - 時間あり: 開始・終了の2行
- ヘッダー行はsticky

### なぜこのUIか

- 同日の複数予定を縦に高速スキャンできるよう、情報密度を優先したため

## 4.6 タスクタブ

### 一覧テーブル

- 列: `案件/カテゴリ` / `タスク` / `ステータス` / `優先度` / `見積` / `実績` / `操作`
- ステータスに応じて行背景色を変更
- 操作:
  - `詳細`
  - `開始`
  - `中断`（`doing`時のみ）
  - `再開`（`suspend`時のみ）
  - `停止`
  - `完了`
  - `削除`

### 登録/詳細モーダル

- 1行目: 案件 / カテゴリ
- 2行目: タスク
- 3行目: 初期ステータス / 優先度
- 4行目: `予定` グループ（開始予定 / 終了予定 / 見積時間）
- 5-6行目: `実績` グループ（開始時間 / 終了時間 / 中断時間 / 実績時間）
- 7行目: 備考

### なぜこのUIか

- 実績管理の中で「中断」を明示的に扱うため、予定と実績を視覚的に分離した

## 4.7 設定モーダル

- 項目:
  - 自動取得時刻
  - 自動取得間隔（値 + 単位）
  - タスク時間表記モード
- 操作:
  - `閉じる` / `未設定にする` / `保存`
- `Esc` / 背景クリックで閉じる

### なぜこのUIか

- 日常操作を阻害しないよう、設定は最小項目に絞り、即時反映ではなく保存反映にした

## 4.8 フッター

- 左: 設定ボタン
- 右: 更新日時 `yyyy/mm/dd HH:mm:ss`

### なぜこのUIか

- 「最後に同期されたか」を常時確認できるようにするため
