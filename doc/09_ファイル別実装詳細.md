# 09. ファイル別実装詳細

## 9.1 Shared

### `src/shared/contracts.ts`

- 全層共有の契約型
- `Task.actual` に `suspendMinutes` / `suspendStartedAt` を定義

### なぜ

- IPC境界で型の二重管理を防ぎ、仕様変更時の破壊を最小化するため

## 9.2 Main

### `src/main/index.ts`

- Composition Root
- Window/Tray/終了動作を統合
- サービスを組み合わせてアプリ全体を起動

### `src/main/googleAuth.ts`

- OAuthフロー
- `userinfo.get()` によるプロフィール取得
- 全カレンダー予定取得（並列 + リトライ）
- ユーザー/トークン保存
- `credentials.json` の探索順制御（環境変数 -> `_shared` 推奨パス -> 互換フォールバック）

### `src/main/appSettings.ts`

- ユーザー別/ゲスト設定の読込・保存
- 値正規化

### `src/main/services/calendarPublisher.ts`

- 予定取得と `calendar-updated` 通知配信

### `src/main/services/autoFetchScheduler.ts`

- 30秒周期で時刻/間隔の実行判定

### `src/main/services/autoLaunch.ts`

- Windows初回自動起動設定

### `src/main/services/taskStore.ts`

- lowdb CRUD
- ユーザー別 `tasks.json` 管理
- 旧データ互換の正規化（`suspendMinutes` 追加対応）
- 案件別候補（カテゴリ/タスク）を動的生成
- 期間集計（`YYYY-MM` / `YYYY`）を生成
  - `projectActuals`（案件別）
  - `categoryActuals`（案件×カテゴリ）
  - `titleActuals`（案件×カテゴリ×タスク）

### `src/main/ipc/registerMainIpcHandlers.ts`

- 機能別IPC登録の合成

### `src/main/ipc/handlers/*.ts`

- `calendarHandlers`: 予定同期IPC
- `settingsHandlers`: 設定IPC
- `taskHandlers`: タスクIPC（一覧、期間集計、追加、更新、削除）
- `authHandlers`: 認証IPC
- `types.ts`: 依存契約型

## 9.3 Preload

### `src/preload/index.ts`

- `window.api` を安全公開

### `src/preload/index.d.ts`

- `window.api` の型定義

### なぜ

- RendererからMainへの許可済み操作だけを公開し、境界を固定するため

## 9.4 Renderer

### `src/renderer/src/App.tsx`

- 全体合成
- 日付シェブロン操作、同期、モーダル接続

### `src/renderer/src/hooks/useAuthController.ts`

- ログイン/ログアウト状態管理

### `src/renderer/src/hooks/useDashboardSettings.ts`

- 設定ドラフトと保存

### `src/renderer/src/hooks/useCalendarRows.ts`

- 予定行状態と通知購読

### `src/renderer/src/hooks/useDateEditor.ts`

- 日付入力の状態・検証・フォーカス制御
- 前日/翌日シフト関数

### `src/renderer/src/components/DateSelector.tsx`

- 日付ボタン、左右シェブロン、入力パネル

### `src/renderer/src/components/ScheduleTable.tsx`

- 予定表テーブル描画

### `src/renderer/src/components/TaskBoard.tsx`

- タスクサブタブ（`タスク一覧` / `タスク集計`）の表示切替
- タスク一覧の検索/フィルタ/ソート/操作
- 期間集計（案件別/カテゴリ別/タスク別）の表示・折りたたみ・条件入力
- 集計/一覧のソート状態管理
- 登録・詳細モーダル
- 実績の中断/再開ロジック連携

### `src/renderer/src/lib/taskBoardSearchSort.ts`

- TaskBoard向けの検索/絞り込み/ソート判定を集約した純粋関数群
- タスク一覧と期間集計の両方で共通利用
- UIコンポーネントからロジックを切り離し、単体テストしやすくする責務を持つ

### `src/renderer/src/components/SettingsModal.tsx`

- 設定入力UI

### `src/renderer/src/components/ProfileSection.tsx`

- ログインUI / プロフィールメニュー

### `src/renderer/src/components/DashboardTabs.tsx`

- タブ切替UI

### `src/renderer/src/lib/dateUtils.ts`

- 日付整形と入力正規化

### `src/renderer/src/lib/settingsUtils.ts`

- 間隔入力の分変換

### `src/renderer/src/lib/taskTimeUtils.ts`

- 見積/実績時間計算
- 昼休憩控除
- 中断時間控除
- 開始/中断/再開/停止処理

### `src/renderer/src/assets/main.css`

- ダッシュボード全体スタイル
- モーダル、テーブル、状態色、レスポンシブ

## 9.6 AI 開発ツール設定ファイル

### `CLAUDE.md`

- Claude Code へのプロジェクトルール・開発フロー・ルール定義

### `.claude/settings.json`

- Claude Code のツール権限設定

### `.claude/skills/electron/` (→ `.agents/skills/electron/` へのシンボリックリンク)

- Electron Skill 定義（agent-browser を使った Electron アプリ操作手順）

### `.agents/skills/electron/SKILL.md`

- Electron Skill の実体ファイル

### `skills-lock.json`

- インストール済み Skill のバージョンロックファイル

### `.devcontainer/Dockerfile`

- Dev Container のベースイメージ定義
- Electron ランタイム依存ライブラリ（`xvfb`、GTK、NSS 等）をプリインストール
- `gh` CLI をプリインストール

### `.devcontainer/devcontainer.json`

- Dev Container の構成定義
- `desktop-lite` 機能（noVNC / ポート 6080）でコンテナ内 GUI を提供
- Named volume で `node_modules` / `gh-config` / `codex-home` / VS Code データを永続化
