# 11. 逸脱変更の棚卸しと是正方針

関連Issue: `#17`, `#25`
基準日: 2026-02-26
比較基準: `origin/main` に対するローカル作業ツリー差分

## 11.1 目的

Issue起点フローを経ずに発生した変更について、対象ファイルごとに採用/撤回を明確化し、以降のPR作業の判断基準を固定する。

## 11.3 実施方針

1. 撤回対象は `origin/main` を正としてローカル差分を解消する。
2. 採用対象は子Issue単位でPRを分けて `development` へ反映する。
3. すべての子Issue完了後に `development -> main` の Release PR で `#17` をクローズする。

## 11.4 最終CI確認（#17 クローズ前）

- 各子Issue PRで `Closes #<子Issue番号>` を記載し、`PR CI (development)` を成功させる。
- すべての子Issueがクローズしたら `development -> main` の Release PR を作成する。
- Release PRで次を確認する。
  - `release-branch-policy` 成功
  - `quality-check` 成功
  - `windows-build` / `macos-build` / `linux-build` 成功
- Release PR本文に `Closes #17` を記載してマージし、親Issueがクローズされることを確認する。
