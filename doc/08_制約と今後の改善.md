# 08. 制約と今後の改善

## 8.1 現状制約

- 永続化は平文 JSON ベース
  - 設定・トークン暗号化は未実装
- カレンダー取得は各カレンダーを逐次取得
  - カレンダー数が多いと待ち時間が増える可能性
- タスクタブはモック実装
  - 実データ連携は未実装
- エラー通知は主に `console.error`
  - ユーザー向けトースト/通知は未整備

## 8.2 セキュリティ観点の改善候補

- `token.json` の保護
  - OS キーストア連携または暗号化
- `credentials.json` の配布運用ルール整理
  - 環境変数/秘密管理基盤への移行
- ログ監査
  - 認証失敗理由や API エラーの構造化ログ

## 8.3 UX 改善候補

- 日付入力パネルに Esc 閉じる対応
- エラーメッセージの多言語化
- タスクタブの CRUD 実装
- カレンダー名フィルタ、件名検索、ソート

## 8.4 アーキテクチャ改善候補

- Google API 取得の並列化とリトライ制御
- IPC のエラーモデル統一
- Renderer のコンポーネント分割
  - `App.tsx` 単一ファイルの責務縮小

## 8.5 テスト戦略の改善候補

- Main ロジックのユニットテスト
  - 設定正規化
  - スケジューラ判定
  - 日付バリデーション
- Renderer コンポーネントテスト
  - 日付入力挙動
  - タブ切替
  - 認証 UI 遷移
